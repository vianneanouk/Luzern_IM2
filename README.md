**Kurzbeschreibung des Projekts**:

Wir haben uns für eine API entschieden, die Besucherzahlen an verschiedenen Orten in Luzern anzeigt. Damit erstellten wir eine interaktive Karte (mit Leaflet), die anzeigt, ob es viele, mässig viele oder wenige Besucher an einem Ort hat. Diese drei Kategorien haben wir selbst erstellt, indem wir die Besucherzahlen der API über ein paar Tage beobachteten. Mehr als 150 Besucher:innen bedeuten viele Besucher, weniger als 50 sind wenige Besucher. Alles dazwischen gehört in die Mittelkategorie. Je nachdem, wie viele Besucher:innen es also hat, wird eine andere Pinfarbe und ein anderer Spruch auf der Karte angezeigt. Zusätzlich wird die exakte Besucherzahl angezeigt und die Zeit, wann zuletzt gemessen wurde.

**Benutzte Ressourcen und Prompts:**

Wir haben den Code mit Hilfe von ChatGPT erstellt. Dabei war uns wichtig, den Code stets zu verstehen. Wir liessen uns einzelne Abschnitte erklären und nahmen gezielt Anpassungen vor. Nicht alle Vorschläge der KI passten zu unserem Ziel, in solchen Fällen haben wir den Code selbst angepasst oder konkret bei den Dozierenden nachgefragt, um ihn richtig umzusetzen. So konnten wir gemeinsam mit der KI eine funktionierende Lösung entwickeln und gleichzeitig unser Verständnis vertiefen.

Ein guter Prompt war: *„Kannst du mir erklären, was in diesem Abschnitt des JavaScript-Codes passiert?“*  Damit erhielten wir verständliche Erklärungen, die uns halfen, den Überblick zu behalten.

Ein schlechter Prompt war: *„Die Spitze der Sprechblase soll am gleichen Ort sein wie die Spitze des Pins.“*  Der war zu ungenau, weshalb die KI falsche oder unpassende Lösungen generierte.
